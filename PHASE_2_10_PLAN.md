# Phase 2-10 快速实现计划

## 策略：MVP优先

每个Phase只实现**核心功能**，跳过美化和边缘case，目标是**快速打通整个游戏流程**。

---

## Phase 2: 捏脸系统（简化版）

**核心功能：**
- [ ] 简单的头像上传（不裁剪，直接显示）
- [ ] 3种预设头像可选
- [ ] 保存到服务器，房间内显示
- [ ] **跳过**：复杂的SVG捏脸、颜色选择器

**时间**：30分钟

---

## Phase 3: 地图与格子系统

**核心功能：**
- [ ] 5×15 格子地图（HTML/CSS Grid）
- [ ] 点击格子高亮
- [ ] 基地和敌人出生点显示
- [ ] **跳过**：PixiJS渲染、相机控制

**时间**：30分钟

---

## Phase 4: 单位系统

**核心功能：**
- [ ] 3种单位：农民、弓箭手、炮塔
- [ ] 点击格子→选择单位→扣金币→部署
- [ ] 农民自动产金币
- [ ] 弓箭手/炮塔攻击逻辑（找最近敌人）
- [ ] 预购确认机制（小袋熊建议）
- [ ] **跳过**：升级系统、复杂动画

**时间**：45分钟

---

## Phase 5: 敌人系统

**核心功能：**
- [ ] 2种敌人：小僵尸、坦克僵尸
- [ ] 从右往左移动（简单线性路径）
- [ ] 受伤、死亡
- [ ] 攻击基地
- [ ] **跳过**：飞行单位、复杂寻路

**时间**：30分钟

---

## Phase 6: 游戏循环

**核心功能：**
- [ ] 金币UI、血量UI
- [ ] 波次系统（10波）
- [ ] 胜利/失败判定
- [ ] 难度选择（甜蜜双排/正常/硬核）
- [ ] **跳过**：暂停、加速

- [ ] 3条路径（上/中/下）

## Phase 10: 双人同步优化

**核心功能：**
- [ ] 游戏状态实时同步（单位、敌人、金币）
- [ ] 基本的断线重连
- [ ] 移动端响应式布局
- [ ] **跳过**：延迟补偿、观战模式

**时间**：30分钟

---

## 总计时间：4.5 小时

**实施策略：**
1. 串行开发，逐个Phase完成
2. 每个Phase commit一次
3. 遇到bug快速绕过（标记TODO）
4. 优先打通流程，不追求完美

**技术简化：**
- 不用PixiJS，改用HTML/CSS + React（更快）
- 动画用CSS transition，不用复杂的帧动画
- 数据结构简化，够用就行

**验收标准：**
- 能玩完整一局游戏
- 双人实时同步正常
- Buff系统生效
- 能通关3关+1个Boss
